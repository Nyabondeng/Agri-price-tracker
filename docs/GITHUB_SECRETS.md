# GitHub Secrets Configuration Guide

This guide explains how to configure GitHub Secrets for the Agri Price Tracker CI/CD pipeline.

## Why GitHub Secrets?

Secrets are encrypted environment variables stored in GitHub. They:
- Are **never logged or displayed** in workflow logs
- Are **only accessible** to authorized workflows
- Support **organization-wide** or **repository-level** scoping
- Rotate easily without changing code

## Prerequisites

1. GitHub repository owner or admin access
2. Terraform outputs from AWS/Azure deployment
3. ACR credentials from Azure Portal
4. SSH private key (generated by Terraform)

## Setup Steps

### Step 1: Navigate to Settings

1. Go to your GitHub repository
2. Click **Settings** (top navigation)
3. Click **Secrets and variables** → **Actions** (left menu)
4. Click **New repository secret**

### Step 2: Configure ACR Secrets (Required for CD)

These secrets are required for the CD pipeline to push Docker images to your private registry.

#### Secret 1: `ACR_LOGIN_SERVER`

**Location**: Azure Portal → Container Registries → Your ACR → Settings → Access keys

**Value**: Login server URL (e.g., `acragricpricetrickeravloqy.azurecr.io`)

**Steps**:
1. In GitHub: Click **New repository secret**
2. Name: `ACR_LOGIN_SERVER`
3. Value: Paste the login server URL from Azure Portal
4. Click **Add secret**

**Verify**: In workflow logs, you should see (secrets are masked):
```
ACR_LOGIN_SERVER=***
```

#### Secret 2: `ACR_USERNAME`

**Location**: Azure Portal → Container Registries → Your ACR → Settings → Access keys → Username

**Value**: ACR username (e.g., `20AllProjects`)

**Steps**:
1. Click **New repository secret**
2. Name: `ACR_USERNAME`
3. Value: Paste username
4. Click **Add secret**

#### Secret 3: `ACR_PASSWORD`

**Location**: Azure Portal → Container Registries → Your ACR → Settings → Access keys → password

**Value**: ACR password (one of the two provided)

**Steps**:
1. Click **New repository secret**
2. Name: `ACR_PASSWORD`
3. Value: Paste password (use password1 or password2, either works)
4. Click **Add secret**

 **Note**: ACR passwords are admin access credentials. Rotate them if compromised:
```bash
az acr credential-renew --name <registry-name> --password-name password1
```

### Step 3: Configure Deployment Secrets (Required for CD VM Deploy)

These secrets enable the CD pipeline to deploy to your Azure VM via Ansible.

#### Secret 4: `BASTION_HOST`

**Location**: Terraform output → `bastion_host_ip`

**Value**: Public IP of Azure Bastion (e.g., `52.123.456.789`)

**Steps**:
1. Get value from Terraform:
   ```bash
   cd terraform
   terraform output bastion_host_ip
   ```
2. Click **New repository secret**
3. Name: `BASTION_HOST`
4. Value: Paste Bastion IP
5. Click **Add secret**

#### Secret 5: `APP_PRIVATE_IP`

**Location**: Terraform output → `app_vm_private_ip`

**Value**: Private IP of Application VM (e.g., `10.0.2.4`)

**Steps**:
1. Get value from Terraform:
   ```bash
   cd terraform
   terraform output app_vm_private_ip
   ```
2. Click **New repository secret**
3. Name: `APP_PRIVATE_IP`
4. Value: Paste App VM private IP
5. Click **Add secret**

#### Secret 6: `SSH_PRIVATE_KEY`

**Location**: File generated by Terraform → `ssh-key-agric-price-tracker.pem`

**Value**: The complete SSH private key (RSA 4096-bit PEM format)

**Steps**:
1. Get the key content:
   ```bash
   cd terraform
   cat ssh-key-agric-price-tracker.pem
   ```
2. Copy the **entire output** including:
   ```
   -----BEGIN RSA PRIVATE KEY-----
   ... content ...
   -----END RSA PRIVATE KEY-----
   ```
3. Click **New repository secret**
4. Name: `SSH_PRIVATE_KEY`
5. Value: Paste the complete private key (multi-line)
6. Click **Add secret**

**Security**: This is the most sensitive secret. Protect it carefully:
- Never share with anyone
- Never commit to Git (use .gitignore)
- Rotate if compromised:
  ```bash
  # In terraform/, regenerate SSH key:
  rm ssh-key-agric-price-tracker.pem
  terraform apply
  # Get new key and update secret
  ```

### Step 4: Optional Azure Authentication Secrets

If using Terraform in CI/CD (optional), configure service principal secrets:

#### Secret 7: `ARM_CLIENT_ID`

**Value**: Azure Service Principal client ID (from Azure AD → App registrations)

#### Secret 8: `ARM_CLIENT_SECRET`

**Value**: Azure Service Principal secret value (from credentials)

#### Secret 9: `ARM_SUBSCRIPTION_ID`

**Value**: Azure subscription ID (from subscription settings)

#### Secret 10: `ARM_TENANT_ID`

**Value**: Azure tenant ID (from directory settings)

**Steps** (same for all):
1. Click **New repository secret**
2. Name: Enter secret name (e.g., `ARM_CLIENT_ID`)
3. Value: Enter the value
4. Click **Add secret**

These are pre-configured in HCP Terraform, so they're optional for repository.

## Summary: Required vs Optional Secrets

### ✅ Required for Image Push (CD Pipeline)
- `ACR_LOGIN_SERVER`
- `ACR_USERNAME`
- `ACR_PASSWORD`

### ✅ Recommended for VM Deployment (CD Pipeline)
- `BASTION_HOST`
- `APP_PRIVATE_IP`
- `SSH_PRIVATE_KEY`

### ⚠️ Optional (Already in HCP Terraform)
- `ARM_CLIENT_ID`
- `ARM_CLIENT_SECRET`
- `ARM_SUBSCRIPTION_ID`
- `ARM_TENANT_ID`

## Verifying Secrets

### List All Secrets
1. Go to GitHub → Settings → Secrets and variables → Actions
2. You should see a list of all created secrets
3. Secrets are masked and show only creation date

### Test Secrets in Workflow

The CD pipeline tests secret availability at the start:

```yaml
- name: Validate ACR and deploy secrets
  env:
    ACR_LOGIN_SERVER: ${{ secrets.ACR_LOGIN_SERVER }}
    # ... other secrets
  run: |
    # Workflow logs output whether secrets are present
    # (don't show actual values)
```

## Updating Secrets

**To change a secret**:
1. Go to GitHub → Settings → Secrets and variables → Actions
2. Find the secret in the list
3. Click the pencil icon to edit
4. Update the value
5. Click **Update secret**

The change takes effect on the next workflow run.

## Rotating ACR Credentials

When ACR passwords expire or need rotation:

```bash
# In Azure CLI:
az acr credential-renew --name <registry-name> --password-name password1

# Then update GitHub secret:
# Settings → Secrets → Edit ACR_PASSWORD → Paste new password
```

## Rotating SSH Key

When SSH key needs regeneration:

```bash
# In terraform/:
rm ssh-key-agric-price-tracker.pem
terraform apply

# Get new key:
cat ssh-key-agric-price-tracker.pem

# Update GitHub secret:
# Settings → Secrets → Edit SSH_PRIVATE_KEY → Paste new key
```

## Troubleshooting

### Issue: "Secrets not available in workflow"

**Symptoms**:
```
Error: Required secret ACR_PASSWORD is empty
```

**Solutions**:
1. Verify secret was created in GitHub (check in Settings)
2. Verify spelling matches exactly (case-sensitive)
3. Verify repo has access to secrets (no branch protection conflict)
4. Wait 1 minute after creating secret; new secrets have propagation delay

### Issue: "SSH connection failed"

**Symptoms**:
```
SSH: Get-Net-TCPConnection -LocalPort 22 timed out
```

**Solutions**:
1. Verify `BASTION_HOST` IP is correct (reachable from public internet)
2. Verify `APP_PRIVATE_IP` is correct (use Terraform output)
3. Verify `SSH_PRIVATE_KEY` matches the private key for Bastion
4. Verify Bastion security group allows SSH (port 22 inbound)

### Issue: "ACR authentication failed"

**Symptoms**:
```
Docker: unauthorized: authentication required
```

**Solutions**:
1. Verify `ACR_LOGIN_SERVER` includes `.azurecr.io`
2. Verify `ACR_USERNAME` and `ACR_PASSWORD` are correct
3. Check credentials haven't expired or been rotated
4. Verify registry is in "normal" state (not locked)

## Security Best Practices

1. **Principle of Least Privilege**: Only add necessary secrets
2. **Regular Rotation**: Rotate ACR passwords every 90 days
3. **Limit Access**: Only admins should modify secrets
4. **Monitor Usage**: GitHub provides action logs (Settings → Actions)
5. **Use Environment Secrets**: For org-wide secrets, use organization-level secrets (Settings → Security)
6. **Audit Trail**: GitHub logs all secret usage (Settings → Audit log)

## Organization-Level Secrets (Optional)

If managing multiple repositories:

1. Go to GitHub Organization → Settings → Secrets and variables → Actions
2. Click **New organization secret**
3. Name the secret (e.g., `ACR_LOGIN_SERVER`)
4. Name: Your repository name
5. Click **Add secret**

This allows repos to reference org-level secrets: `${{ secrets.ACR_LOGIN_SERVER }}`

## Referencing Secrets in Workflows

Secrets are referenced in GitHub Actions workflows using this syntax:

```yaml
- name: Example step
  env:
    SECRET_NAME: ${{ secrets.SECRET_NAME }}
  run: echo $SECRET_NAME  # Outputs: ***
```

The secret value is:
- ✅ Available to the step
- ✅ Masked in logs (shows ***)
- ❌ Never logged or displayed
- ❌ Cannot be captured or exported

## Cleanup

If you no longer need a secret:

1. Go to GitHub Settings → Secrets and variables → Actions
2. Find the secret
3. Click the trash icon
4. Confirm deletion

The secret is immediately unavailable to new workflow runs.
